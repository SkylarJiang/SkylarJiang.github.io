Cookie

Posted on 2020-07-20 Edited on 2020-07-25 Valine:

0 Cookie 是什么

  • 是服务器发送到用户浏览器上的一小段文本文件，浏览器会进行储存然后随着下次的请求一起发回给服务器，Web 服务器就可以使用这些信息来识别 不同的用户。

  • 是纯文本格式，不含任何可执行的代码

  • 使用目的：

    • Session 管理：登录、购物车、游戏进度等
    • 个性化（Personalization）
    • 用户跟踪（Tracking）：记录分析用户行为

1 创建 Cookie

Set-Cookie： HTTP 响应头发送给用户，信息头

Cookie：浏览器在收到 Set-Cookie 之后的每一次请求头都要把所有显示存储的 cookies 一起发回去

cookie 中可以指定任意数量的选项，顺序任意

修改和覆盖

name = value 是 cookie 的值，其他只是附加的属性而已

想要改变 Cookie 的值，就必须发送一个所有选项值都相同的 Set - Cookie 信息头，不然就会新建一个

会有两个同名的 Cookie , path 越详细越靠前

2 Cookie 编码

原始规范：必须进行编码的只有：; , = ；可以进行 URL 编码，但不是必须

几乎有实现都对 Cookie 值进行了一系列 URL 编码，对于 name=Vvalue 格式，通常会对 name 和 value 进行 URL 编码，而不对 = 进行编码

3 支持选项

  1. Expires & Max-age
  2. Secure：只在 HTTPS 时发送
  3. HTTPOnly：规定不能使用 JavaScript
  4. Domain & Path
  5. SameSite

3.1 Lifetime ：Expires & Max-Age

没有设置 expires 选项时，cookie 的生命周期仅限于当前会话。登录 Web 应用时询问是否记住登录信息就是选择是否把 expires 附加到登录 Cookie 中

改变一个 cookie 的失效日期需要指定相同的组合

改变 cookie 的值时

Max-age 是指有效时间有多长，Expires 是到某个时间点失效

HttpOnly无法通过document.cookie（禁止通过js访问）访问

Domain 可以访问到上级域名中的 cookie，某个路径下的所有二级域名都能访问到这个 cookie

不同域名不能共享

常用cookie来保存 session

定位到 用户
