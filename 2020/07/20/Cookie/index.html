Cookie

Posted on 2020-07-20 Edited on 2020-07-25 Valine:

Cookie 的创建、选项、维护等内容的小结。

[TOC]

0 Cookie 是什么

  • 是服务器发送到用户浏览器上的一小段文本文件，浏览器会进行储存然后随着下次的请求一起发回给服务器，Web 服务器就可以使用这些信息来识别不同的用户。

  • 是纯文本格式，不含任何可执行的代码

  • 使用目的：

    • Session 管理：登录、购物车、游戏进度等
    • 个性化（Personalization）
    • 用户跟踪（Tracking）：记录分析用户行为

1 创建 Cookie

Set-Cookie： HTTP 响应头发送给用户，信息头

Cookie：浏览器在收到 Set-Cookie 之后的每一次请求头都要把所有显示存储的 cookies 一起发回去

2 Cookie 编码

原始规范：必须进行编码的只有：; , = ；可以进行 URL 编码，但不是必须

几乎有实现都对 Cookie 值进行了一系列 URL 编码，对于 name=Vvalue 格式，通常会对 name 和 value 进行 URL 编码，而不对 = 进行编码

3 支持选项

  1. Expires & Max-age
  2. Secure & HTTPOnly
  3. Domain & Path
  4. SameSite

3.1 Lifetime ：Expires & Max-Age

Max-age 是指有效时间有多长，Expires 是到某个时间点失效

如果 expires 设置了一个过去的时间点，或者Max-Age 为 0，那么这个 cookie 会立即被删除

没有设置 expires 和Max-Age 选项时，cookie 的生命周期仅限于当前会话。

登录 Web 应用时询问是否记住登录信息就是选择是否把 expires 附加到登录 Cookie 中。

3.2 Restrict access:

  • HTTPOnly:

设置 HttpOnly 后无法通过 document.cookie（禁止通过 JavaScript 访问）访问，当然也没办法通过 JavaScript 来进行 HTTPOnly 选项的设置。

设计该特征意在提供一个安全措施来帮助阻止通过 JavaScript 发起的跨站脚本攻击 (XSS) 窃取 cookie 的行为。

  • secure

当一个请求通过 SSL 或 HTTPS 创建时，包含 secure 选项的 cookie 才能被发送至服务器。默认情况下，在 HTTPS 连接上传输的 cookie 都会被自动添加上 secure 选项。

以纯文本方式进行传递很有可能被篡改。

  • SameSite

cookie 不允许跨域请求，帮助阻止伪造跨域请求攻击 (CSRF)

Strict Lax none

3.3 Domain & Path

Domain 和 Path 制定了 cookie 的作用域：即 cookie 应该发送给那些 URL

  • Domain：

    • 标识制定了哪些主机可以接收 cookie，如果不指定，默认为当前主机（不包含子域名）

    • 扩充 cookie 可发送域的数量：

      如 name.yahoo.com 形式的某个站点把其 cookie 的 domain 选项设置为 yahoo.com 就可以将该 cookie 的值发送至所有这些站点。

    ​ domain 的值根请求的域名的尾部比较

  • Path

    • 选项指定了请求的资源 URL 中必须存在指定的路径时，才会发送Cookie 消息头

    • 这个比较通常是将 path 选项的值与请求的 URL 从头开始逐字符比较完成的。

      如果设置 Path=/blog 也会匹配 /blogrool ，任何以 /blog 开头的选项都是合法的

只有在 domain 选项核实完毕之后才会对 path 属性进行比较。

维护

  • 选项：

    cookie 中可以指定任意数量的选项，顺序任意

  • 修改和覆盖：

    想要改变 cookie 的值，必须发送另一个所有标识符都相同的 Set-Cookie 消息头；如果修改 cookie 选项中任意一项都将创建一个新的 cookie

  • Cookie 中的排序：同名的 cookie，按照domain-path-secure的顺序，值越详细则越靠前

  • Expires 和 Max-Age 不是表示符的一部分，所以修改 cookie 的值的时候不需要重新制定过期时间

自动删除

  • Session cookie 在会话结束时被删除
  • 持久化 cookie (Persistent Cookie) 到达失效日期
  • 浏览器中 cookie 数量达到限制

cookie 的限制条件

  • cookie 的属性
  • cookie 的数量限制
  • cookie 的总大小：发向服务器的所有 cookie 的最大数量（空间）仍旧维持原始规范中所指出的：4KB。

Subcookies

subcookies 观点：增加 cookie 的存储量。允许单个 cookie 中保存多个 name-value对

操作

在 JavaScript 中通过 document.cookie 属性，你可以创建、维护和删除 cookie。创建 cookie 时该属性等同于 Set-Cookie 消息头，而在读取 cookie 时则等同于 Cookie 消息头。

注意：一旦 cookie 通过 JavaScript 设置后便不能提取它的选项，所以你将不能知道 domain，path，expires 日期或 secure 标记。

Cookie & Session

常用cookie来保存 session

在网站开发中，最常用的就是用 cookie 来保存 session，但是二者并不等价。cookie 是存在客户端，session 是存在服务器端。服务端根据客户发过来的 cookie（eg.cookie 中有 session_id）定位到这个客户的信息（session），给他提供最合适的响应
